# FinalAlarm — 기획 정리

## 1. 컨셉

팀 기반 알람 앱. 혼자서는 알람을 끌 수 없고, 같은 팀에 묶인 다른 팀원이 풀어줘야 꺼지는 구조.

- Android (Kotlin) 먼저 → iOS (SwiftUI) 추후
- 서버는 기존 인프라에 도커 컨테이너로 합류

## 2. 핵심 기능

### 알람 종류 (두 가지 공존)
- **팀원 승인 알람**: 팀원이 승인해야 끌 수 있는 알람
- **일반 알람**: 혼자서 끌 수 있는 일반 알람 (Alarmy 같은 일반 알람앱처럼)

### 알람 끄기 = 사용자가 사전 설정한 미션
- 본인이 미리 끄는 미션을 선택해서 등록해둠
- 지원 미션: **수학 문제 / 사진 인증 (QR·바코드 포함) / 휴대폰 흔들기**
- 팀원 승인 알람은 **팀원이 승인 → 사용자가 본인 설정 미션 수행** 2단계
- 일반 알람은 **사용자가 본인 설정 미션을 바로 수행**

### 알람 발사자 표시
- 알람을 받은 사람에게 **누가 보냈는지 항상 공개** (익명 없음)

### 팀 소속
- 한 사람이 **여러 팀 동시 소속 가능** (가족 / 친구 / 회사 등)
- 알람 발사 / 시간대 / 알람 공유 등은 팀 단위로 동작

### 잠금해제 요청 플로우 (팀원 승인 알람)
1. 알람이 울림 → 사용자가 "끄기 요청" 전송
2. 같은 팀 멤버에게 요청 푸시 알림 도착
3. 팀원이 승인하면 → 해당 알람이 "잠금해제 가능" 상태로 전환
4. 사용자가 본인이 설정한 방식(비밀번호/미션)으로 끄기 수행
5. **요청 만료: 5분** — 팀원 응답 없으면 만료

### 팀 알람 가시성
- **팀원 승인 알람**: 같은 팀 멤버 모두에게 공개 (서로의 알람을 조회 가능)
- **일반 알람**: 본인만 조회 가능 (팀에 공개되지 않음)

### 알람 시간대 (Alarm Window)
- 본인이 "이 시간대 안에는 누가 알람 보내도 됨" 시간 범위 설정
- **여러 개 설정 가능, 겹쳐도 각자 별개로 유지** (머지하지 않음)
- 그 시간대 내에서:
  - 팀원 누구나 그 사람에게 알람을 발사할 수 있음
  - 본인도 자기 알람을 설정 가능
  - 한 번 끄고 나서도 시간대 내라면 또 다른 알람이 트리거될 수 있음 (반복 가능)

### 팀 초대
- **초대 코드** + **초대 링크(딥링크)** 둘 다 지원
- 초대 코드: 6~8자리, 만료시간 있음
- 초대 링크: 딥링크/Universal Link, 탭 시 앱 자동 가입 흐름

## 3. 플랫폼 정책 / 기술 제약

### Android — 거의 다 됨
- `USE_EXACT_ALARM` (Android 12+): 알람앱이면 Play Console에서 자동 승인
- 원격 알람 트리거: **FCM high-priority 메시지**로 Doze 모드 우회
- 알람음 재생: **Foreground Service**
- 비밀번호 잠금 해제 방식: 정책 제약 없음 (Alarmy 등 선례 다수)
- ⚠️ **제조사별 배터리 최적화 우회** UI 필수 (샤오미/삼성 등이 백그라운드 종료시킴)

### iOS — 제약 있지만 Alarmy 방식으로 간다
- **Critical Alerts entitlement는 사용하지 않음** (확정)
  - Apple 심사 거절률 높음 + Alarmy도 미사용 확인됨
  - 무음/방해금지 모드 우회 포기 — 사용자에게 안내로 대체
- 접근: **Alarmy 방식** — 백그라운드 오디오 세션 + 로컬 알림 조합
  - 앱이 백그라운드/포어그라운드일 때: 백그라운드 오디오로 알람음 재생
  - 앱이 종료되어 있을 때: 로컬 알림 사운드(30초 제한)로 fallback
  - 사용자에게 "이 앱을 백그라운드에서 유지하고, 무음/방해금지 끄세요" 안내

## 4. 서버 아키텍처

- **별도 git 레포**로 운영. coachDesk와 같은 호스트에 각자 docker-compose로 독립 실행
- coachDesk PostgreSQL 공유 — 외부 네트워크 join, 테이블 prefix `alarm_*`로 격리
- **백그라운드 워커 없음** — 알람 발사·만료·스누즈는 클라이언트 책임 + lazy expiration
- FCM은 API 핸들러에서 동기 호출 + 자동 재시도
- 푸시 토큰 관리, 팀/알람/이벤트 CRUD 등 전부 REST API

## 5. 진행 순서

1. **데이터 모델 + API 설계** (양 플랫폼 공유) ← 다음 단계
   - User, Team, Membership
   - Alarm, AlarmWindow
   - UnlockRequest (잠금해제 요청)
   - PushToken
2. **서버 구현** (도커 컨테이너, FCM 연동)
3. **Android 앱 구현**
   - 스캐폴딩 (Compose, Room, FCM, Hilt 등)
   - 핵심 플로우 PoC: FCM 수신 → Foreground Service 알람 → 비밀번호 화면
   - 전체 화면/플로우
4. **iOS 앱 구현** (Critical Alerts 신청은 1~2단계에서 미리 넣기)

## 6. 결정 사항 로그

- ✅ 알람 종류: 팀원 승인 알람 + 일반 알람 둘 다 지원
- ✅ 끄기 미션: 수학 문제 / 사진 인증(QR·바코드 포함) / 휴대폰 흔들기
- ✅ 일반 알람 공유: 본인만 (팀에 노출 안 됨)
- ✅ 팀원 승인 알람 공유: 팀 전체 공개
- ✅ 팀 초대: 초대 코드 + 초대 링크 둘 다
- ✅ 알람 시간대: 여러 개 가능, 겹쳐도 별개
- ✅ 잠금해제 요청 만료: 5분
- ✅ 발사자 표시: 항상 공개 (익명 없음)
- ✅ 다중 팀: 한 사람이 여러 팀 소속 가능

## 7. 기본 알람앱 기능 (전부 포함)

이 앱은 "팀 기능 + 기본 알람앱 기능"을 모두 제공. 즉, 사용자가 이 앱 하나로 OS 기본 알람을 대체 가능해야 함.

- ✅ 일회성 알람 / 반복 알람 (요일 선택)
- ✅ 알람 사운드 (시스템 + 사용자 음악) — **알람별 설정**
- ✅ 진동 (on/off + 패턴 선택) — **알람별 설정**
- ✅ 볼륨 + 점진적 증가 (volume ramp)
- ✅ 스누즈 (on/off, 분 단위, 최대 횟수)
- ✅ 라벨/이름
- ✅ 활성·비활성 토글

## 8. 데이터 보존

- 알람 히스토리: 무기한 유지. **admin 페이지에서 수동 정리** (6개월 이상 삭제 버튼)
- 자동 보존 정리 워커 안 만듦 — 관리 요소 줄이는 게 우선

## 9. 추후 정책 (MVP 이후)

- 알람 무한 반복 방지 (시간대 내 N번 제한, 쿨다운 등) — 일단 무제한, 베타로 사용해보고 정책 결정
- 팀 최대 인원수 제한 — 일단 무제한, 운영하며 조정
